                		|

-Fazer um menu onde seja possível o usuário escolher se quer:
	.1_Incluir um novo cadastro;
	.2_Consultar um cadastro;
	.3_Excluir um cadastro;
	.4_Créditos; e
	.5_Sair do sistema.


-Solicitar que o usuário escolha uma destas opções.
	*Caso o usuário escolha "1_Incluir um novo cadastro", o sistema deve validar da seguinte forma:
		.Verifica se já existem cadastros:
			-Se existir cadastros anteriores:
				.Verifica para onde aponta o ultimo ponteiro do ultimo cadastro realizado:
					-Se o cadastro apontar para outro e que não esteja seguindo uma ordem (Exemplo:1,2,3,5...):
						.O sistema aloca um espaço entre os dois registros em desordem (3 e 5), com valor que os deixe em uma ordem (no caso do exemplo o novo local teria COD com valor 4)
					-Se o cadastro estiver com o ponteiro NULL, 
						.O sistema deve alocar uma nova posição(conforme o caso da estrutura de ponteiros mostrada acima)
			-Se não existir nenhum cadastro:			
				.Iniciar o identificador (COD) em 1, fazendo com que a cada novo registro seja incrementado este COD. Não pode haver CODs iguais;
					-Solicitar que o usuário digite os dados cadastrais:
						.Nome;
						.Endereço;
						.Telefone;
					-Fazer a inclusão destes dados na primeira estrutura, utilizando alocação dinâmica;
					-Criar uma nova lista a partir do ponteiro, conforme ilustrado no caso "Fazer a Estrutura";
					-Incrementar o identificador e inicializar o ponteiro em null na nova lista. Conforme ilustrado no caso "Fazer a estrutura".
			-Após incluir, o sistema volta ao menu principal.

	
	*Caso o usuário  escolha "2_Consultar um cadastro", o sistema deve validar da seguinte forma:
		.Solicitar que o usuário informe o COD do cadastro que deseja consultar;
		.Verifica o COD informado pelo usuário:
			-Se o COD informado não existir, 
				.Retornar mensagem dizendo: COD informado não existe.
			-Se o COD informado for uma letra ou caractere especial, 
				.Retornar mensagem dizendo: COD informado não existe.
			-Se o COD informado for válido e existir, 
				.retornar as informações referentes a este COD.
			-Utilizar o COD informado pelo usuário para localizar o cadastro que procura. Imprimindo este cadastro na tela;
		.Após consultar, o sistema volta ao menu principal.
+MELHORIA: Caso sobre tempo ou consigamos fazer, validar a busca pelo nome também e não apenas pelo COD.


	*Caso o usuário escolha "3_Exlcuir um cadastro", o sistema deve validar da seguinte forma:
		.Solicitar que o usuário informe o COD do cadastro que deseja excluir;
		.Verifica o COD informado pelo usuário:
			-Se o COD informado não existir, 
				.Retornar mensagem dizendo: COD informado não existe.
			-Se o COD informado for uma letra ou caractere especial, 
				.Retornar mensagem dizendo: COD informado não existe.
			-Se o COD informado for válido e existir,
				.Utilizar o COD informado pelo usuário para localizar o cadastro que procura;
					.Verificar se o ponteiro que está nesta lista está com com valor NULL ou não.
						.Caso esteja NULL, 
							-Fazer a exclusão sem problema algum;
						.Caso esteja preenchido, 
							-O ponteiro da lista anterior deve apontar para o ponteiro da lista que está para ser excluída;
				.Utilizar o comando FREE para liberar a memória do ponteiro que aponta para a estrutura que deve ser excluída.
+MELHORIA: Caso sobre tempo ou consigamos fazer, validar a exclusão pelo nome também e não apenas pelo COD.

	
	*Caso o usuário escolha "4_Créditos", o sistema deve validar da seguinte forma:
		.Direcionar o usuário a uma tela onde seja exibido os dados dos colaboradores deste trabalho.


	*Caso o usuário escolha "5_Sair do sistema", o sistema deve validar da seguinte forma:
		.O programa deve ser finalizado.
